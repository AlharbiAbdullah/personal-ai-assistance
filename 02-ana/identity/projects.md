**Last updated:** 2026-01-01

## Active

### open-kit
An open-source command-line toolkit for developers — scaffolding, repo hygiene, and small automations I got tired of rewriting. The flagship side project; the one I most want to get right.
- **Stage:** building toward v1.0
- **Why:** a useful, well-crafted OSS tool is the best portfolio piece there is.

### taskflow
A small SaaS experiment — a focused task/planning tool for solo makers. The vehicle for learning the whole product loop: billing, auth, support, retention.
- **Stage:** prototype → first users
- **Why:** I want to understand the full path from code to paying customer.

### cloud-lab
A home lab for learning infrastructure hands-on — self-hosting, containers, a bit of Kubernetes, and a place to run my own services.
- **Stage:** ongoing, low-pressure
- **Why:** the best way to learn ops is to run something you'd miss if it broke.

---

## Rules for what makes the list

1. It has a clear next action.
2. I'd genuinely use or miss it.
3. It teaches me something I want to know.

If a project can't clear those three, it goes back to [[ideas]] or dies.
