# 04-work/ — John's Work Footprint

Daily-job artifacts: projects, meetings, tasks, proposals, diagrams. Anything tied
to a paid engagement.

A project can exist in both `04-work/` AND `05-projects/` when it's both paid and
personal — overlap is allowed. This folder is the work view.

## Structure
- `{engagement}/` — one subfolder per engagement (e.g., `client-alpha/`, `tableau/`). All
  engagement files live inside.
- `work-plans/` — weekly plans named by ISO week (`2026-W17.md`).

## Rules
- Confidentiality first. Contains client/employer details. Never share externally
  without explicit authorization.
- No AI/Claude mentions in any output that lands here.
- Diagrams: Excalidraw for editable, PDFs for finalized.
- New engagement → new subfolder. Short obvious name.

## Skills
The `/work` skill group lives in `03-rai/skills/work/`:
- `weekly-planner` — Monday ritual; reads last week's plan + `02-ana/identity/goals.md` + open tasks
- `meeting-prep` — pre-meeting briefing and agenda

## What Claude does
- Default to formal tone unless told otherwise.
- Verify proprietary names/amounts/dates with John. Do not infer.
- Never auto-share, post, or upload work content.
- When unsure if something is shareable: NO by default, ask.
- When the user says "work file" or "the proposal" — look here first.
