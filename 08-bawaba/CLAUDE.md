# 08-bawaba/ — Daily / Weekly News Digests

## Purpose

Personalized news digests generated by the `NewsDigest` skill (`/news`). Sources include Hacker News, Reddit, X (Twitter), Substack, GitHub Trending.

## What lives here

- `daily/YYYY-MM-DD.md` — daily digests (auto-generated 3am via systemd)
- `weekly/YYYY-WWW.md` — **Bawaba Weekly**, the magazine issue for that Sun–Sat week (auto-generated Saturday 7am). Not a recap: topical departments in magazine prose — Editor's Letter, Cover Story, Model State, The Lesson, The Workshop, Reading Shelf, Closing Wisdom. Mined from the week's raw dumps (`13-archive/news/dumps/`) + dailies, web-enriched. This is the most important output of the news system.

Prior digests are archived automatically after each run to `13-archive/news/` (`daily/` and `weekly/` subfolders).
- `_graduated.md` — items promoted out of news into `09-ideas/` or elsewhere
- `story-arcs.md` — multi-day narrative threads
- `news_x_*.json` and similar — scratch capture files (gitignored)

## Rules

- **Generated content only.** This folder is the output target of the `/news` skill. Manual edits are unusual.
- **Digests are immutable after creation.** A daily digest reflects what was relevant on that date. Do not retroactively edit.
- **Gaps are okay.** If a day was skipped, leave the gap. Do not backfill.
- **Scratch JSONs are debug artifacts.** Covered by `.gitignore` rules. Safe to delete after the skill finishes.

## Automation

Both digests run unattended via systemd user timers on the Ubuntu box (`03-rai/skills/news-digest/scheduled/run-news-ubuntu.sh`, headless `claude -p`):
- **Daily** — 3am (`news-daily.timer`), drives the local logged-in Chrome for X/Substack/Medium, output to `daily/`.
- **Weekly** — Saturday 7am (`news-weekly.timer`), no browser: runs `weekly_mine.py` over the week's dumps, reads the dailies, enriches subjects via WebSearch/WebFetch, writes the magazine to `weekly/`.

## What Claude should do

When the user says "what's happening" or "/news" or "give me a digest":
- Invoke the `/news` (NewsDigest) skill.
- Do not hand-write digests in this folder.

When the user references a past digest by date, read the file directly.

When the user wants to "promote" a news item into a real idea or knowledge note:
- Append a line to `_graduated.md` with the source digest date and destination.
- Move the substantive content to `09-ideas/` (as a Seed) or `10-knowledge/` (as a topic note).
