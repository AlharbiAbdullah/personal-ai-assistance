# Annual Reports

Index last updated: 2026-Q1. Maintained in `references/reports-index.json`.

Aggregate and analyze cybersecurity annual reports from 570+ sources.
Search across reports by category, year, or topic. Extract trends,
statistics, and actionable intelligence.

## Categories

| Category | Coverage |
|----------|----------|
| Threat Intelligence | APT groups, campaigns, TTPs |
| Vulnerabilities | CVE trends, exploitation timelines |
| Ransomware | Groups, payments, targets, evolution |
| AI / Emerging Tech | AI-powered attacks, LLM risks, deepfakes |
| Cloud Security | Misconfigurations, identity attacks |
| Industry Reports | Sector-specific threat landscapes |

Category overlaps: Threat Intelligence covers APT groups, campaigns, TTPs. Vulnerabilities covers CVE trends and patch timelines. When in doubt, check both.

Trends require year-over-year comparison. Single-year reports show a snapshot, not a trend. If comparing, pull the same category from adjacent years.

Search: "Find reports on [topic]" or "Show [category] reports for [year range]". Rai searches the index by keyword + category + year range.

## Process

1. **Identify scope**: Topic, year range, category filter
2. **Search reports**: Match against the report index
3. **Extract findings**: Pull relevant statistics, trends, quotes
4. **Cross-reference**: Compare findings across multiple reports
5. **Synthesize**: Produce a summary with citations to source reports

## Output Format

- Lead with key findings (3-5 bullet points)
- Support each finding with data from specific reports
- Note the report name, publisher, and year for each citation
- Flag conflicting data between sources
- End with trend direction (increasing/decreasing/stable)

## Examples

- "What are the top ransomware trends from 2025 reports?"
- "Search annual reports for AI-related threats"
- "Compare vulnerability exploitation timelines across reports"
- "What do 2025 reports say about cloud security posture?"
- "Find threat intel on APT groups targeting healthcare"
